#### Как работает оплата

Оплата происходит через встроенный магазин Tilda (T-Store) на сайте [business46.ru](https://business46.ru).

**Схема:**
```
Бот (кнопка «Перейти к оплате») → redirect-страница на сервере → сайт business46.ru с popup-корзиной Tilda
```

1. Пользователь выбирает тариф в боте → бот показывает inline-кнопку «Перейти к оплате»
2. Кнопка ведёт на redirect-страницу (см. ниже), которая перенаправляет на сайт с нужным товаром
3. На сайте открывается popup-корзина Tilda с полями: Имя, Фамилия, Email, Телефон
4. После заполнения и оплаты через Tilda — пользователь пишет боту «оплатил» или отправляет скриншот
5. Админ подтверждает или отклоняет оплату

#### Redirect-страницы (почему не прямые ссылки)

Tilda использует hash-фрагменты для открытия корзины: `business46.ru/#order:Название=Цена`. Telegram обрезает hash-фрагменты с кириллицей из URL-кнопок, поэтому прямые ссылки не работают.

Решение — промежуточные HTML-страницы на сервере, которые делают JavaScript-редирект:

| Тариф | Redirect-URL (в боте) | Куда перенаправляет |
|-------|----------------------|---------------------|
| Standard | `https://ubuyuedekrut.beget.app/pay/standard.html` | `business46.ru/#order:Билет на конференцию=4990` |
| Premium | `https://ubuyuedekrut.beget.app/pay/premium.html` | `business46.ru/#order:Конференция + AfterParty=6790` |

**Файлы на сервере:** `/var/www/html/pay/standard.html` и `/var/www/html/pay/premium.html`
**Nginx:** в конфиге `/etc/nginx/sites-enabled/nodejs` добавлен `location ^~ /pay/` для раздачи этих файлов как статики.

#### Если нужно изменить цену или название товара

1. Изменить цену/название на сайте в Tilda (Магазин → Товары)
2. Обновить redirect-страницы на сервере:

3. Обновить тексты в боте (`src/texts.js`) если нужно изменить отображение цен
4. Перезапустить бота: `pm2 restart conf-bot`

#### Как найти правильную ссылку на товар Tilda

Если товар изменился или добавился новый:
1. Открыть `business46.ru/#rec1925094411` (блок с тарифами)
2. Правой кнопкой мыши по кнопке «ПРИНЯТЬ УЧАСТИЕ» нужного тарифа → «Просмотреть код»
3. В HTML найти атрибут `href` — например `href="#order:Конференция + AfterParty=6790"`

## Статистика

### `/stats` — воронка

Показывает полную воронку: /start → сегмент → программа → тарифы → выбор тарифа → оплата.
Включает разбивку по сегментам, тарифам, выручку, точку наибольшей потери.

### `/stuck` — застрявшие пользователи

Три группы для ручного дожима:
1. **Выбрали тариф, не оплатили** — самые горячие
2. **Смотрели тарифы, не выбрали** — тёплые
3. **Выбрали сегмент, дальше не пошли** — холодные

### Ежедневный отчёт

Автоматически отправляется в 20:00 МСК админу статистики (5419539623).

## State management

Поле `state` в БД определяет, как бот обрабатывает текстовые сообщения:

| State | Значение | Реакция на текст |
|-------|----------|-----------------|
| `new` | Только нажал /start | «Используй кнопки» |
| `segmented` | Выбрал сегмент | «Используй кнопки» |
| `awaiting_question` | Нажал «Есть вопрос» | Пересылает текст админу |
| `awaiting_payment` | Выбрал тариф | «оплатил» / фото → пересылает админу |
| `paid` | Оплата подтверждена | «Используй кнопки» |

Приоритет обработки текста:
1. `awaiting_question` → question.js
2. `awaiting_payment` + «оплатил»/фото → checkout.js
3. Всё остальное → «Используй кнопки»

## Установка и запуск

### Локально (разработка)

```bash
cd bot
cp .env.example .env
# Заполнить .env (токен, ID админов, ссылки на оплату)
npm install
npm run dev    # с авторестартом при изменениях
```

### На сервере (продакшн)

```bash
ssh -i ~/.ssh/beget_vps root@155.212.139.76
cd /opt/apps/conf-bot
# .env уже настроен
npm install
pm2 start ecosystem.config.js
pm2 save
```

### Обновление на сервере

Файлы загружаются через SCP, затем перезапуск PM2:

